# discreet_graph_lab
Висновки та порівняння алгоритмів Крускала і Флойда–Воршала

У роботі було реалізовано та експериментально досліджено два класичні алгоритми теорії графів: алгоритм Крускала для знаходження мінімального остовного дерева та алгоритм Флойда–Воршала для знаходження найкоротших шляхів між усіма парами вершин. Для кожного алгоритму було згенеровано серію випадкових зв’язних графів різного розміру (10, 20, 50, 100, 200, 500 вершин) та з різною заповненістю (ймовірність ребра p = 0.2 та p = 0.5). Для кожного набору параметрів алгоритми запускалися кілька разів, після чого фіксувався мінімальний час виконання.

Експерименти показали, що для алгоритму Крускала час виконання зростає зі збільшенням кількості вершин і густини графа, що відповідає теоретичній складності O(E log E), оскільки зі зростанням p зростає кількість ребер, які необхідно сортувати. Реалізований власноруч алгоритм Крускала виявився швидшим за вбудовану реалізацію NetworkX. Це пояснюється меншими накладними витратами: власна реалізація працює з простішими структурами даних і повертає лише список ребер, тоді як NetworkX створює повноцінний граф MST та виконує додаткові універсальні перевірки.

Для алгоритму Флойда–Воршала спостерігалося значно швидше зростання часу виконання зі збільшенням розміру графа. Це відповідає його кубічній складності O(n³), через що алгоритм стає практично непридатним для великих графів. Навіть при помірних значеннях n час виконання швидко зростає, незалежно від заповненості графа. Порівняння з реалізацією NetworkX показало, що бібліотечна версія працює стабільно швидше, що пов’язано з внутрішніми оптимізаціями та реалізацією на більш низькому рівні.

Коректність власних реалізацій була перевірена шляхом порівняння результатів з результатами NetworkX. Для алгоритму Крускала порівнювалася сумарна вага мінімального остовного дерева, яка в усіх тестах збігалася з результатом NetworkX. Для алгоритму Флойда–Воршала порівнювалися матриці найкоротших відстаней, отримані власною реалізацією та бібліотечною функцією nx.floyd_warshall, і результати збігалися для випадкових тестових графів.

Отже, експерименти підтвердили як коректність реалізованих алгоритмів, так і відповідність емпіричної складності теоретичним оцінкам. Алгоритм Крускала є ефективним для побудови MST навіть для відносно великих графів, тоді як алгоритм Флойда–Воршала доцільно застосовувати лише для невеликих графів через його високу обчислювальну складність. Порівняння з NetworkX показало, що бібліотечні реалізації є більш універсальними та оптимізованими, однак власні реалізації є корисними для навчальних цілей і глибшого розуміння принципів роботи алгоритмів.
